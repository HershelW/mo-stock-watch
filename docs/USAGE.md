# 使用说明

## 启动

推荐日常使用：

```powershell
cargo run --release
```

也可以双击：

- `start-mo-stock-watch.bat`
- `run-release.bat`

第一次编译会比较慢，之后会复用 `target/` 编译缓存。

## 持仓维护

1. 点击 `工具` 展开工具栏。
2. 点击 `编辑` 进入编辑模式。
3. 可添加、删除或修改持仓行。
4. 点击 `保存` 写入本地数据文件。
5. 退出编辑后会继续按刷新间隔拉取行情。

持仓字段：

- `代码`：6 位 A 股代码。
- `名称`：显示名称，不影响行情查询。
- `数量`：持仓股数。
- `成本`：成本价。
- `市场`：根据代码自动推断，也可手动选择沪/深/北。

## 行情刷新

应用会从行情接口拉取数据。正常刷新间隔可在工具栏调整，最小 5 秒。刷新失败时会自动延迟重试，避免频繁请求。

当前行情源顺序：

1. 东方财富
2. 腾讯
3. 新浪

每次刷新会先请求东方财富；如果失败、超时或返回空数据，会自动尝试腾讯；腾讯也失败时再尝试新浪。状态栏会显示本次成功使用的行情源。

为了减少无意义请求，应用只会在工作日交易相关时段请求行情：

- `09:20-11:30`
- `13:00-15:00`

其他时间不会请求东方财富接口，会在状态栏显示距离下次刷新还有多久。当前版本只按本机日期判断工作日和周末，暂未接入法定节假日/临时休市日历。

## 数据迁移

持仓和设置在：

```text
%APPDATA%\mo-stock-watch\
```

换电脑时复制这个目录即可。源码目录只负责程序本身，不保存个人持仓。

## 常见问题

### 项目目录为什么很大？

大头通常是 `target/`，这是 Rust 编译缓存和构建产物，不是源码。它已被 `.gitignore` 排除，不会上传 GitHub。

清理方式：

```powershell
cargo clean
```

### 行情一直失败怎么办？

先确认网络可以访问东方财富接口。如果只是临时失败，应用会自动重试。若接口字段变化，需要更新 `src/quote.rs`。

### 保存后移动项目会不会丢数据？

不会。数据在 `%APPDATA%\mo-stock-watch\`，不是项目目录。
